⚙️ 第三章

你开口的瞬间
发生了什么？

从你按下回车到 Claude 读到上下文，中间只有几十毫秒

1
你按下发送
就在这一刻，Claude Code 的 UserPromptSubmit hook 悄悄被触发了——就像古代击鼓传令兵，在你的话真正传出去之前，先跑了一圈收情报
2
gather.sh 多路采集
像一座自动气象站， gather.sh 同时启动所有启用的传感器：读系统时间、查 wttr.in 天气、问 git 分支状态
3
config.json 决定采集什么
config.json 就是驾驶舱仪表盘——哪个指示灯是亮的，gather.sh 就采集哪路数据，天气默认关闭减少延迟
4
打包成 JSON 注入
采集到的信息被组装成一段 JSON，告诉 Claude Code 把它放进 additionalContext ——就像导演在演员上场前悄悄塞一张小纸条：「今晚是周五，下雨，她刚失业」
5
Claude 生成回复
Claude 读到你的消息时，这些上下文已经在了。它知道现在几点、你在哪个分支、外面什么天气，然后才开始回答你
⚙️ 技术描述
UserPromptSubmit hook fires on message submit
bash gather.sh reads config.json + runs subcommands
curl wttr.in?format=3 with 3s timeout
Output JSON: hookSpecificOutput.additionalContext
Claude Code injects context before LLM call
🌏 通俗中文
你发消息的瞬间，一个钩子被悄悄拉起来
脚本读配置文件，决定要采集哪些信息
用 curl 问 wttr.in 天气，3秒内没答就跳过
把信息打包成 JSON，告诉 Claude Code 注进去
Claude 读消息前，上下文已经在了

📋 你装好了 claude-context-injector，但每次和 Claude 说话，它还是不知道你在哪个 Git 分支。你检查了 config.json，发现是这样的：
{"time": true, "git": false, "weather": false}
问题出在哪，怎么修？
A 重新运行 install.sh，重装一次
B 把 "git": false 改成 "git": true，保存后重启 Claude Code
C 每次对话前手动告诉 Claude 分支名
🌤️ 你开启了天气功能，但发现每次发消息都要多等 3 秒，因为 wttr.in 有时响应很慢。你怎么既保留天气功能，又减少等待？
A 把 weather 设为 false，彻底关掉天气
B 没办法，天气就是慢
C 在 gather.sh 里降低 curl 的超时时间（当前是 3 秒），比如改为 1 秒，超时就跳过
🔧 你想给 gather.sh 加一个新功能：注入当前系统内存使用率。你会怎么做？

B 在 gather.sh 里仿照现有数据源加几行，在 config.json 里加对应的开关
C 修改 ~/.claude/settings.json 里的 hook 命令
